# calls.md — UserCalls (the call/meeting metadata layer)

A **UserCall** is the user-facing wrapper around a recording. Verified shape against prod (this DTO is *deeply nested* — much more so than the catalog's schema summary suggests):

```json
{
  "id": 2294,                              // outer integer ID — use this in /usercall/{id} paths
  "workspaceId": "f9873261-…",
  "clickupList": null,                     // populated when this call has been pushed to ClickUp
  "jiraProjectId": null,                   // ditto for Jira
  "jiraCloudId": null,
  "airtableBaseId": null,
  "airtableTableId": null,
  "summaryApplied": false,                 // booleans — has the summary been pushed somewhere?
  "actionItemsApplied": false,
  "callSpecificCustomPrompt": null,
  "tags": [],
  "viewSummaries": [],
  "call": {
    "id": 2789,                            // INNER id — Zoom/Meet/Teams call id; NOT the path id
    "callId": "4b9d0e7b-…",
    "callUuid": "04b82b61-…",              // <— this is what selection endpoints want
    "description": "",
    "attendees": "",                       // STRING, not a list of {email,name}
    "startTime": "2026-04-06T13:24:17Z",
    "processed": false,
    "recordings": [
      {
        "id": 2630,                        // recording integer id — use in /recording/{id}/...
        "transcription": "Actually, now let's…",
        "summary": "# Heading\n**Summary**…",     // markdown STRING
        "keyPoints": "- Focus of discussion: …",  // markdown STRING (bullets), NOT array
        "actionItems": "Here are the action items mentioned:\n\n1. …",  // markdown STRING
        "createdAt": "…",
        "meetingEndedAt": "…",
        "changedTasks": [ /* Hub's auto-generated task changes */ ]
      }
    ]
  }
}
```

### Three identifiers — get them right

The DTO has **two integer ids and one UUID**, easy to confuse:

| Field | Where | What it's for |
|---|---|---|
| `id` (outer) | top level | Path argument for `GET /usercall/{id}`, `DELETE /usercall/{id}`. THE ONE you usually want. |
| `call.id` (inner) | nested | Internal Zoom/Meet/Teams call ID. **NOT** valid in `/usercall/{id}` paths — passing it 404s. |
| `call.callUuid` (string) | nested | The `callUUID` body field for `POST /usercall/selection/{jira,clickup,airtable}`. |

When iterating search results: `outer.id` for the path, `outer.call.callUuid` for the action pushes. Recording IDs come from `outer.call.recordings[i].id`.

### Text fields are markdown strings, not structured objects

`summary`, `keyPoints`, `actionItems` are all markdown-formatted **strings** generated by Lens. There is no structured array of `{description, assignee, dueDate}` for action items — if you want structure, you have to parse the markdown yourself.

`attendees` is also a string (often empty).

## Listing + searching (read)

All require `recordings:read` scope.

### List most-recent calls

```
GET /api/usercall/all?page=0&size=20
```

Returns `[]` of `UserCallDTO`. Page + size pagination; defaults documented in the catalog. Most-recent first.

### Total count

```
GET /api/usercall/all/count
```

Returns a bare integer (e.g. `123`). Pair with `/all` for paginated UIs.

### Free-text search

```
GET /api/usercall/search?query=acme&page=0&size=20
```

Searches across call name, description, attendees, AND recording transcript content. Returns `[]` of matching calls. Empty `query=` returns all calls (same as `/all`).

### Search count

```
GET /api/usercall/search/count?query=acme
```

Bare integer. Pair with `/search` for paginated UIs.

### Get one

```
GET /api/usercall/{id}      # use the OUTER id from list/search responses, not call.id
```

Returns full `UserCallDTO` (shape documented at the top of this file).

This is the primary "what did this call cover?" call. Read `call.recordings[0].summary` for an executive overview, `call.recordings[0].actionItems` for what to push to a task system, and `call.recordings[i].id` for the integer recording ID needed by `/recording/{id}/audio-url` etc.

## Mutating (write)

All require `recordings:write` scope.

### Delete a call

```
DELETE /api/usercall/{id}
```

**Hard-deletes both the UserCall metadata AND the underlying recording media. Not reversible.** Returns `Response.ok(true)` on success. Confirm with the user before calling this — even when the user says "delete the call about X", consider listing first and showing them what will be deleted.

### Send action items to Jira

```
POST /api/usercall/selection/jira
Content-Type: application/json

{
  "callUUID": "<uuid from get_call.call.callUuid>",
  "cloudId": "<from list_integrations Jira entry>",
  "projectIds": ["ENG", "OPS"]
}
```

Hub maps each action item to a Jira issue under the listed projects. `projectIds` is a `Set<String>` — pass an array even when there's only one project. Body field is `projectIds` (plural), NOT `projectId`.

Requires:
- `recordings:write` scope
- A Jira integration on the workspace (`GET /v1/workspaces/{id}/integrations` shows it)

**Important**: returns boolean — `true` means **the request was accepted**, NOT that it actually landed in Jira. Hub records the selection and a downstream worker performs the push asynchronously (and silently fails on bad cloudId / project / vendor token expiry). Verified during smoke: posting a deliberately invalid `selectedListId` to the ClickUp variant still returned `true` HTTP 200. **Don't tell the user "pushed to Jira" based on this boolean alone** — surface as "queued for push" or follow up by checking the call's `clickupList`/`jiraProjectId`/`airtableBaseId` field on a fresh `GET /usercall/{id}` to confirm.

### Send action items to ClickUp

```
POST /api/usercall/selection/clickup
Content-Type: application/json

{
  "callUUID": "<uuid from get_call.call.callUuid>",
  "selectedListId": "<ClickUp list id>"
}
```

Single list per call. Same auth + return semantics as Jira (boolean response = "request accepted", not "actually pushed").

### Send action items to Airtable

```
POST /api/usercall/selection/airtable
Content-Type: application/json

{
  "callUUID": "<uuid from get_call.call.callUuid>",
  "baseId": "<Airtable base id>",
  "tableIds": ["tbl_xxx", "tbl_yyy"]
}
```

`tableIds` is a `Set<String>` — Hub fans the action items out across the listed tables in the same base. Same auth + return semantics (boolean response = "request accepted", not "actually pushed").

### Backfill metadata

```
POST /api/usercall/fill
```

Internal-style endpoint used by Hub's meeting-processing pipeline to enrich metadata. Rarely needed by external callers; the catalog lists it but you generally won't use it. If you do, the request shape is `UserCallFillRequest` — fetch the catalog detail.

## Common patterns

### Find a call by description and push its actions to Jira

```python
import requests, os

KEY = os.environ["DUTIFY_HUB_API_KEY"]
H   = {"X-API-Key": KEY}

# 1. Discover the bound workspace UUID (one HTTP call — Hub filters /v1/workspaces
#    to the bound workspace for API-key callers).
ws_list = requests.get("https://dutify.ai/api/v1/workspaces", headers=H).json()
WS = ws_list["workspaces"][0]["id"]

# 2. Find the call
calls = requests.get(
    "https://dutify.ai/api/usercall/search",
    headers=H, params={"query": "acme renewal"},
).json()
call = calls[0]   # most-recent match

# 3. Resolve cloudId — this DTO doesn't expose vendor-specific fields anymore,
#    so the user has to provide it or you have to look it up out of band.
#    Just confirm Jira is connected:
integrations = requests.get(
    f"https://dutify.ai/api/v1/workspaces/{WS}/integrations",
    headers=H,
).json()
if not any(i["integrationSystem"] == "JIRA" and i["status"] == "CONNECTED" for i in integrations):
    raise RuntimeError("Jira not connected on this workspace.")

cloud_id = os.environ["JIRA_CLOUD_ID"]   # supplied by user / external config

# 4. Push action items
resp = requests.post(
    "https://dutify.ai/api/usercall/selection/jira",
    headers={**H, "Content-Type": "application/json"},
    json={
        "callUUID":  call["call"]["callUuid"],   # NESTED — call.callUuid, not top-level
        "cloudId":   cloud_id,
        "projectIds": ["ENG"],                   # array, even for one project
    },
)
# resp.json() is `true` if Hub accepted the request — NOT a guarantee it landed in Jira.
# To confirm landing, fetch the call again and check its top-level jiraProjectId field
# is populated.
print(resp.json())
```

### Iterate every visible call

Cursor pagination doesn't exist here — page through with `page` + `size` until you get fewer results than `size`:

```python
page = 0
size = 100
while True:
    batch = requests.get(
        "https://dutify.ai/api/usercall/all",
        headers=H, params={"page": page, "size": size},
    ).json()
    if not batch:
        break
    for c in batch:
        process(c)
    if len(batch) < size:
        break
    page += 1
```

For a planning UI where you want a total before iterating, hit `/api/usercall/all/count` first.
